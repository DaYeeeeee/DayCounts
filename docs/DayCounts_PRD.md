# DayCounts 产品需求文档（PRD）

更新时间：2026-04-27  
项目路径：`D:\Qt_Projects\DayCounts`

## 1. 文档目标

本 PRD 基于当前仓库实现进行整理，目标是回答 3 个问题：

1. 当前项目已经具备哪些能力。
2. 这个项目接下来最值得实现哪些功能。
3. 现有界面还可以从哪些方向优化，才能从“原型”走向“可用产品”。

---

## 2. 当前项目现状

### 2.1 已实现内容

根据当前代码，项目已经完成了桌面应用的基础外壳：

- 使用 Qt Quick / QML 构建主界面。
- 自定义无边框窗口。
- 已实现最小化、最大化、关闭按钮。
- 左侧有一个可悬停展开的侧边区域。
- 中间区域已有月历网格，可展示当前月份日期。
- 支持点击日期进行单日选择。

### 2.2 当前代码对应的模块

- `main.cpp`：应用启动入口。
- `main.qml`：加载主窗口。
- `qml/DCWindow/MainPage.qml`：主窗口容器，负责整体布局。
- `qml/DCWindow/TitleButton.qml`：标题栏右上角窗口控制按钮。
- `qml/DCWindow/LeftPage.qml`：左侧可展开区域。
- `qml/DCWindow/CalendarPage.qml`：基础月历视图。
- `qml/DCWindow/BottomPage.qml`：当前为空，尚未承载业务内容。

### 2.3 当前版本判断

当前版本更接近“UI 原型”而不是完整产品，原因如下：

- 没有“倒数日/纪念日”数据模型。
- 没有新增、编辑、删除事件能力。
- 没有本地持久化存储。
- 没有倒数结果展示卡片。
- 没有提醒、分类、筛选、搜索等真实使用能力。
- 左侧栏和底部区域尚未承载有效导航或业务信息。

## 3. 产品定位建议

### 3.1 建议定位

DayCounts 可以定位为：

“一个聚焦纪念日、目标日、待办节点与生活事件管理的桌面倒数日应用”

### 3.2 目标用户

- 需要记录生日、纪念日、考试日、发薪日、出行日的个人用户。
- 希望每天打开电脑就能看到近期重要日期的人群。
- 喜欢轻量桌面工具、而不想依赖复杂日程系统的用户。

### 3.3 核心价值

- 一眼看到“还有多少天”。
- 帮用户记住重要日期。
- 用日历和列表两种方式组织时间事件。
- 提供简单、美观、低学习成本的桌面体验。

---

## 4. 核心使用场景

### 场景 1：记录重要日期

用户添加“考试日 2026-06-20”，首页立即显示“距离考试还有 54 天”。

### 场景 2：查看近期提醒

用户打开应用，能快速看到未来 7 天或 30 天内即将到来的事件。

### 场景 3：按日历管理事件

用户在月历中点选某一天，查看当天有哪些纪念日、任务节点或倒计时事件。

### 场景 4：区分不同类型事件

用户将事件分为“工作 / 学习 / 生活 / 纪念日”，便于筛选和视觉识别。

## 5. 功能规划建议

## 5.1 P0：从原型变成可用产品的必做功能

### 1. 倒数日事件管理

- 新增事件。
- 编辑事件。
- 删除事件。
- 支持事件名称、目标日期、分类、备注、颜色。

这是整个产品最核心的能力，优先级最高。

### 2. 倒数计算与展示

- 展示“距离某日还有 X 天”。
- 展示“某事件已经过去 X 天”。
- 支持按最近到期排序。
- 支持在首页展示重点事件卡片。

### 3. 本地数据持久化

- 关闭应用后数据仍保留。
- 建议首版使用 JSON、SQLite 或 `QSettings`。

如果没有持久化，应用无法形成日常使用闭环。

### 4. 首页总览页

建议新增一个真正的主页面，而不是只显示日历：

- 今日日期与星期。
- 最近 3 到 5 个重要事件。
- 今日已过/剩余天数概览。
- 快速新增入口。

### 5. 日历联动事件列表

- 点击某一天后，右侧或下方显示该日期对应事件。
- 区分“当天发生的事件”和“未来目标事件”。

### 6. 左侧导航栏

当前左侧栏只有展开效果，没有内容。建议补齐导航：

- 首页
- 日历
- 全部事件
- 分类
- 设置

## 5.2 P1：增强体验的高价值功能

### 1. 事件分类与标签

- 预设分类：纪念日、生日、工作、学习、旅行。
- 支持自定义颜色标签。

### 2. 搜索、筛选、排序

- 按名称搜索。
- 按分类筛选。
- 按最近到期、创建时间、字母顺序排序。

### 3. 重复事件规则

- 每年重复：生日、纪念日。
- 每月重复：账单、发薪日。
- 每周重复：例行提醒。

### 4. 提醒能力

- 距离事件 7 天、3 天、1 天提醒。
- 到期当天提醒。

### 5. 空状态与引导

- 首次使用时显示引导文案。
- 没有事件时展示“去添加第一个重要日子”。

## 5.3 P2：适合后续版本的进阶功能

### 1. 数据导入导出

- 导出 JSON。
- 导入备份。

### 2. 多视图切换

- 卡片视图。
- 列表视图。
- 时间轴视图。

### 3. 桌面小组件/悬浮卡片

- 固定显示最近一个倒数日。
- 可选透明小窗模式。

### 4. 云同步

- 多设备同步事件数据。

### 5. 节日与农历扩展

- 中国节日预置。
- 农历生日或纪念日支持。

## 6. 界面优化建议

## 6.1 当前界面问题总结

当前界面已经有一定视觉基础，但还存在这些问题：

- 主体信息层级不明确，不容易一眼看出“核心信息是什么”。
- 左侧栏只有动效，没有导航语义。
- 日历只有日期网格，没有月份标题、切换按钮、选中态强化和事件标记。
- 主内容区较空，缺少倒数卡片、列表、详情面板。
- 色彩比较单一，暂时更像练习界面，不像成熟工具产品。

## 6.2 主窗口布局优化

建议将主界面拆成三层：

### 第一层：标题栏

- 保留自定义窗口按钮。
- 增加应用名称。
- 增加拖动区域。
- 可增加“新增事件”快捷按钮。

### 第二层：左侧导航

当前 `LeftPage.qml` 适合继续演进成导航栏：

- 收起态显示图标。
- 展开态显示图标 + 文本。
- 当前选中项高亮。
- 底部放设置入口。

### 第三层：内容区

根据导航切换不同页面：

- 首页总览页。
- 日历页。
- 事件列表页。
- 设置页。

## 6.3 日历界面优化

当前 `CalendarPage.qml` 建议重点优化：

### 1. 增加月份头部

- 显示“2026 April”或“2026年4月”。
- 增加上个月/下个月按钮。
- 支持快速回到今天。

### 2. 优化日期格子状态

- 今天高亮。
- 选中日期高亮。
- 非当前月日期置灰，而不是完全留空或使用高对比色块。
- 悬停时增加轻反馈。

### 3. 增加事件提示

- 有事件的日期格子显示小圆点或数量角标。
- 点击后右侧显示当天事件详情。

### 4. 丰富信息密度

- 当前月事件数量统计。
- 最近到期事件预览。

## 6.4 首页总览页优化

建议新增独立首页，并作为默认进入页：

- 顶部显示今天日期与一句状态文案。
- 中间显示最重要的 1 个主卡片倒数。
- 下方显示“最近到期”“今天相关”“本月重要日”三个分区。

这样产品打开后就能体现“DayCounts”的主题，而不是直接落到一个普通月历。

## 6.5 事件列表页优化

建议新增列表页，满足管理需求：

- 每行显示事件名、目标日期、剩余天数、分类标签。
- 支持编辑和删除。
- 支持按剩余天数排序。
- 支持搜索框。

## 6.6 新增/编辑事件弹窗优化

建议使用轻量弹窗或右侧抽屉：

- 事件名称输入框。
- 日期选择器。
- 分类选择。
- 颜色选择。
- 备注输入。
- 是否重复。
- 是否提醒。

## 6.7 视觉风格优化

建议方向：

- 用更克制的主色，而不是整块高饱和粉蓝背景。
- 提高卡片层级，通过浅阴影、圆角、分区留白体现信息组织。
- 文本颜色做深浅分层，避免纯白文字大面积使用。
- 保留圆角和轻柔风格，走“温和、治愈、效率工具”路线。

## 10. 结论

这个项目的方向是成立的，现阶段最缺的不是更多动画，而是把“倒数日产品核心闭环”补齐。

优先顺序建议如下：

1. 先做事件数据模型和本地存储。
2. 再做首页总览和事件管理。
3. 然后补日历联动、分类、筛选和提醒。
4. 最后再做更高级的视觉打磨和进阶能力。

如果按这个顺序推进，DayCounts 可以比较顺畅地从当前 UI 原型升级成一个真正可交付、可持续迭代的桌面产品。
